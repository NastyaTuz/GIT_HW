1. git checkout .
2. git restore --staged 
3. git reset HEAD^ ( кол-во ‘^’ зависит от того, на сколько коммитов нужно откатиться назад)
4. git revert HEAD

5. Командой git branch «Название ветки» мы создаем новую ветку, для переключения ветки нужна команда git checkout «Название ветки», git branch -v - показывает все созданные ветки

5. Fast-forward-merge - быстрый способ слияния ( Переключиться на ветку в которую будем вливать , команда Git merge + название которую надо вливать)
 Non-fast-forward merge- имеет место быть когда есть параллельные изменения в целевой ветке, и тогда нужно создать новый коммит, чтобы объединить изменения из двух веток. 

6.конфликт слияния веток может случиться когда две ветки изменены в одной и той же части файла или в одном и том же файле, которые не могут быть объединены автоматически.
Чтобы разрешить конфликт нужно открыть файл в котором произошел конфликт, откроется текстовый редактор, в нем нужно удалить ненужную информацию и удалить маркеры конфликта, сохранить и выйти из редактора. Выполнить git commit.

7. Удаление ветки командой Git branch -d ‘Название ветки’ (перед удалением, убедиться что находимся на не удаляемой ветке)
